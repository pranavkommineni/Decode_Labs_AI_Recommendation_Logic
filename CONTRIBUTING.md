# Contributing to AI Career Recommendation Engine

First off, thank you for your interest in contributing to the AI Career Recommendation Engine! 🎉

This project aims to provide intelligent, explainable, and personalized career recommendations using Artificial Intelligence and Information Retrieval techniques.

We welcome contributions that improve functionality, performance, documentation, security, and user experience.

---

## Code of Conduct

By participating in this project, you agree to:

- Be respectful and professional.
- Provide constructive feedback.
- Follow coding best practices.
- Maintain a collaborative environment.
- Respect the opinions and ideas of other contributors.

---

## How to Contribute

### 1. Fork the Repository

Click the **Fork** button on GitHub and create your own copy of the repository.

### 2. Clone the Repository

```bash
git clone https://github.com/your-username/AI-Career-Recommendation-Engine.git

cd AI-Career-Recommendation-Engine
```

### 3. Create a Feature Branch

```bash
git checkout -b feature/new-feature
```

### 4. Make Changes

Implement your feature, improvement, or bug fix while following the project structure and coding standards.

### 5. Test Your Changes

Ensure that:

- The application runs correctly.
- APIs function as expected.
- No existing functionality is broken.
- Documentation is updated if required.

### 6. Commit Changes

```bash
git add .

git commit -m "Add: Brief description of changes"
```

### 7. Push Changes

```bash
git push origin feature/new-feature
```

### 8. Submit a Pull Request

Open a Pull Request describing:

- What was changed
- Why it was changed
- Screenshots (if applicable)
- Related issues

## Coding Standards

### Python

- Follow PEP 8 guidelines.
- Use meaningful variable names.
- Add comments where necessary.
- Write reusable and modular code.

### Frontend

- Maintain clean HTML structure.
- Use readable CSS classes.
- Keep JavaScript modular.

## Reporting Bugs

When reporting bugs, please include:

- Operating System
- Browser (if frontend issue)
- Python Version
- Steps to reproduce
- Expected behavior
- Actual behavior
- Error messages or screenshots
